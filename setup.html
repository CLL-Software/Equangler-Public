Equangular embed setup

Version 2.6.7 (V2.6.7). One script tag per page; equation/table divs from the builder.

1. Latest release (@main)

Tracks the newest build. URL stays the same across releases; content may change when main is updated.

<script src="https://cdn.jsdelivr.net/gh/CLL-Software/Equangler-Public@main/eqn.js"></script>
ProsCons
Simplest snippet; always current fixes on mainNo integrity hash; updates without a version bump

2. Version-pinned (recommended for production)

Immutable V2.6.7 release. Use Subresource Integrity so the browser rejects tampered files.

<script
  src="https://cdn.jsdelivr.net/gh/CLL-Software/Equangler-Public@V2.6.7/eqn-secure.js"
  integrity="sha384-vWgyLCANxhODeUOvUb9prAvu4Ko+JXEFgFp/mCF/pLPNWDeSa3t+w5zgahaGD6i0"
  crossorigin="anonymous"></script>
ProsCons
Fixed behaviour; SRI verification; obfuscated bundleMust update tag + hash for each release

3. Self-hosted (download or build locally)

Host eqn.js or eqn-secure.js on your own origin instead of jsDelivr.

Download the latest build directly:
https://raw.githubusercontent.com/CLL-Software/Equangler-Public/refs/heads/main/eqn.js

This is a temporary download location and may be replaced with release-specific download links in the future. For production deployments, consider downloading a version-pinned release asset instead when available.

Or build from source (Equangular repo):

node embed/build-production.js

Then upload embed/dist/eqn.js or embed/dist/eqn-secure.js to your server.

<script src="/assets/eqn-secure.js"
  integrity="sha384-vWgyLCANxhODeUOvUb9prAvu4Ko+JXEFgFp/mCF/pLPNWDeSa3t+w5zgahaGD6i0"
  crossorigin="anonymous"></script>
ProsCons
No third-party CDN dependency; full deployment controlYou are responsible for updates and security patching
